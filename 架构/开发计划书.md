## 一、 商品详情模块 (Product Detail Module)

- **目标：** 实现用户查看商品详细信息的功能，包括图片展示、描述、价格、相关规格以及加入购物车/立即购买操作。
- **依赖：** 零.2 数据库准备 (确保 `product.detail`, `goods.ch_spec` 字段存在)。

### 1.1 后端实现 (`ProductDetailServlet.java` - 新建)

- **技术考量：**
  - 错误处理：对无效的商品 ID、数据库查询失败等情况进行妥善处理。
  - 日志记录：关键步骤加入日志记录，便于调试。
  - 服务层抽象：数据库操作应通过 DAO 层和服务层进行。
- [ ] 创建 `src/com/yang/controller/ProductDetailServlet.java` 文件。
- [ ] 定义 `@WebServlet("/productDetail")` 注解。
- [ ] 实现 `doGet` 方法：
  - [ ] 步骤 1: 获取请求参数 `id` (商品 ID)，进行非空和格式校验。
    - **验证：** 单元测试校验不同 ID 输入（有效、无效、空）。
  - [ ] 步骤 2: 调用服务层方法，根据 `id` 查询 `Goods` 对象 (`currentGoods`)。
    - **技术提示：** 需要确保 `IGoodsService` (或类似接口) 有相应方法，如 `getGoodsById(int id)`。如没有，需先在 `GoodsDaoImpl` 和 `GoodsServiceImpl` 中添加。
    - **验证：** 单元测试模拟服务层返回，或集成测试验证正确获取商品。
  - [ ] 步骤 3: 通过 `currentGoods.getProduct_id()` 调用服务层获取 `Product` 对象，并从中提取 `detail` 文本 (`productDetailText`)。
    - **技术提示：** 需要服务层方法支持通过 `product_id` 查询 `Product`。
    - **待讨论点 (RFC 212):** 明确 `product.detail` 中详细描述和花语的提取规则。暂时假定 `product.detail` 直接包含这两部分，后续可细化。
    - **验证：** 单元测试/集成测试验证 `productDetailText` 获取正确。
  - [ ] 步骤 4: 构建附加图片 URL 列表 (`additionalImages`)。
    - 逻辑：根据 `currentGoods.getImg()` (主图路径，例如 `xh_img1.jpg`)，在 `web/images/` 目录下查找 `xh_img1_1.jpg`, `xh_img1_2.jpg` 等附加图片。
    - **技术提示：** 需要文件系统访问逻辑，注意路径拼接和文件存在性检查。
    - **验证：** 准备测试图片，单元测试/集成测试验证 `additionalImages` 列表构建正确。
  - [ ] 步骤 5: 使用 `currentGoods.getProduct_id()` 调用服务层查询 `goods` 表，获取相关规格商品列表 (`relatedGoodsList`)。
    - **技术提示：** 服务层需要方法如 `getGoodsByProductId(String productId)`。
    - **验证：** 单元测试/集成测试验证 `relatedGoodsList` 获取正确。
  - [ ] 步骤 6: 将 `currentGoods`, `productDetailText`, `additionalImages`, `relatedGoodsList` 设置为 `request` 属性。
    - **验证：** 单元测试验证 `request` 属性设置正确。
  - [ ] 步骤 7: 请求转发到 `productDetail.jsp`。
    - **验证：** 集成测试验证页面正确跳转。
- [ ] 编写 `ProductDetailServlet` 的单元测试。

### 1.2 前端实现 (`web/productDetail.jsp` - 新建)

- **技术考量：**
  - JSTL/EL：充分利用 JSTL 和 EL 表达式展示后端传递的数据。
  - CSS：样式尽量与现有商城风格保持一致，或定义新的模块化样式。
  - JavaScript：用于图片切换和数量调整，代码应清晰易维护。
- [ ] 创建 `web/productDetail.jsp` 文件。
- [ ] 页面整体布局设计 (参考 RFC 1.2)：
  - [ ] 引入必要的 CSS (如 Bootstrap) 和 JS (如 jQuery)。
  - [ ] 左侧：商品图片展示区。
    - [ ] 主图显示区域：使用 `<img>` 标签，默认显示 `currentGoods.img`。
    - [ ] 缩略图列表：遍历 `additionalImages` 列表 (以及主图本身)，生成缩略图 `<img>` 标签列表。
  - [ ] 右侧：商品信息与操作区。
    - [ ] 商品核心信息：显示 `currentGoods.name`, `currentGoods.price`, `productDetailText` (含描述和花语)。
    - [ ] 相关规格选择区：遍历 `relatedGoodsList`。
      - [ ] 每个规格项显示图片 (`relatedGood.img`) 和中文规格描述 (`relatedGood.ch_spec`)。
      - [ ] 每个规格项链接到 `/productDetail?id=${relatedGood.id}`。
    - [ ] 底部操作区 (数量选择器，“加入购物车”，“立即购买”按钮)。
      - [ ] 数量输入框 (默认为 1)，增加/减少按钮。
      - [ ] “加入购物车”按钮。
      - [ ] “立即购买”按钮。
- [ ] 实现客户端 JavaScript 交互：
  - [ ] 图片切换逻辑：点击缩略图，更新主图区域的 `src` 属性。
    - **验证：** 手动测试图片切换功能。
  - [ ] 数量调整逻辑：点击增加/减少按钮，更新数量输入框的值，确保不小于 1。
    - **验证：** 手动测试数量调整功能。
- [ ] 表单/链接处理：
  - [ ] “加入购物车”按钮：点击后，构造 GET 请求 `/session?id=${currentGoods.id}&num=${selected_quantity}`。
    - **验证：** 点击按钮，检查浏览器地址栏或网络请求。
  - [ ] “立即购买”按钮：点击后，构造 GET 请求 `/session?id=${currentGoods.id}&num=${selected_quantity}&buyNow=true`。
    - **验证：** 点击按钮，检查浏览器地址栏或网络请求。
- [ ] **可测试性：** 确保 JSP 页面元素有合适的 ID 或 class，便于未来可能的自动化 UI 测试。
- [ ] **验证：**
  - 部署后，通过浏览器访问不同商品的详情页，检查所有信息是否按预期展示。
  - 图片切换、数量调整、规格切换链接、加入购物车、立即购买按钮功能符合 RFC 描述。
